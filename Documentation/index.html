  • Scroll animationos: Reveal

    Reveal animations: there are two types of reveal animations:

    1. Cover animations:cover_animation();
    2. Fade move animations:fade_move();
    3. Blur text animations:blur_text_animation();
    4. SVG animations:svg_draw_animation();

    You can use the first 2 of these animations on endless number of HTML elements and the 3rd one on Endless texts and the 4th on SVGs.

    To use each animation style you need to add 2 classes to some related elements; one to a parent element which the animation starts when it comes into view and the other one to the element itself which you are going to animate on scroll.

    The parent class name could be anything that you like but the element itself must have the same class as the parent +__elat the end of it.

    For the first 2 functions class name +__elwill animate an entire block, you can also animate letters, words; to do so use these instead of__eluse:__letters,__wordsand__lineshere is an example:

    You can change some of the functions directly inside the HTML, this includes: delays, offsets, and paddings. Just add data attributes to the parent element.

    The data attributes which you can use inside HTML for the first 2 functions are:

    • data-whatEverAnimationNameIs-delay: adds delay in seconds.
    • data-whatEverAnimationNameIs-offset: changes the starting point position in pixels, can have negative amounts.
    • data-trigger-hook: means which part of the parent element should cpme to view in order to start the animation. can have one of these values:
      1. onEnter
      2. onCnter
      3. onLeave
    • data-pt: addpadding-topin pixels.
    • data-pr: addpadding-rightin pixels.
    • data-pb: addpadding-bottomin pixels.
    • data-pl: addpadding-leftin pixels.

    The data attributes which you can use inside HTML for the 3rd functionblur_text_animation();are:

    • data-whatEverAnimationNameIs-time: adds animation running time in seconds.
    • data-whatEverAnimationNameIs-stagger: adds stagger time in seconds.
    • data-whatEverAnimationNameIs-delay: adds delay in seconds.
    • data-whatEverAnimationNameIs-offset: changes the starting point position in pixels, can have negative amounts.

    The data attributes which you can use inside HTML for the 4th functionsvg_draw_animation();are:

    • data-whatEverAnimationNameIs-delay: adds delay in seconds.
    • data-whatEverAnimationNameIs-offset: changes the starting point position in pixels, can have negative amounts.

  • Scroll animationos: Intro

    There are 13 different styles for loading screen. each demo has a different style. some of them have special CSS styles and HTML structure.

    The HTML structure should be at the end of the page.

    A class should be added to<html>element for each loading screen style. The classes are:.animation-loading--01,.animation-loading--02all the way up to.animation-loading--13

    After the loading screen goes away, elements of the page can have landing animations. You can apply each of these animations on an endless number of elements.

    It works exactly like reveal animations:cover_animation();andfade_move();. In fact, they are exactly those functions, except for that they run when the loading screen goes away.

    So everything is like before, for any more information head back to Reveal animations.

    The simplest HTML structure for loading screen is like this:

    Here we explain the data-attributes:

    1. data-attributes on<div id="loading-screen" .. > .. </div>:
      • data-opacity: determines how the fading animation of#loading-screenwill be when it starts to go away. "1" means it wont fade away.
      • data-y-percent: determines how thetranslateYanimation of#loading-screenwill be when it goes away, in percents
      • data-time-last-watch: Imagine that the loading process has been finished. Now you like to have some time to watch the elements inside the#loading-screenbefore the it goes away. This is that time in miliseconds
      • data-time-page_els_overtake: If you choose for the elements of the page to animate in after the#loading-screengoes away; you can specify this time which will allow those elements to start animating in even before the#loading-screenstarts to go away. so that the whole animation will be more consistent. in miliseconds of course.

    2. data-attributes on<div class="loader" .. > .. </div>:
      • data-top: determines how thetopanimation of.loaderwill be when it goes away
      • data-opacity: determines how the fading animation of.loaderwill be when it starts to go away. "1" means it wont fade away.

    3. data-attributes on<div class="infinito__word" .. >on assumption that there is an SVG element there:
      • data-drawing-time-sec: determines the time of the SVG draw animation for each of the SVG parts.
      • data-total-drawing-time-milisec: determines the cumulative time of the SVG draw animation for all of SVG parts. all together
      • data-stagger-time-sec: determines the stagger time of the SVG draw animation for each of the SVG parts.

    4. data-attributes on<div id="pace-local-styles" .. > .. </div>, these will apply on<div class="pace" .. > .. </div>element:
      • data-top: determines how thetopanimation of.pacewill be when it goes away.
      • data-scale-x: determines how thescaleYanimation of.pacewill be when it goes away.
      • data-scale-y: determines how thescaleYanimation of.pacewill be when it goes away.
      • data-opacity: determines how the fading animation of.pacewill be when it starts to go away. "1" means it wont fade away.
